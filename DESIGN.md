## 1. ภาพรวมและเป้าหมาย

### แนวคิดหลัก
บอร์ดเกมดิจิทัลแบบ "เดินช่อง" (roll-and-move) ที่ผู้เล่น 2–4 คน (หรือแบ่งทีม) ทอยลูกเต๋าเดินหมากไปตามเส้นทาง เพื่อเก็บ **เหรียญราชภักดิ์** และเรียนรู้ประวัติศาสตร์ผ่านช่องคำถาม ช่องภารกิจ ช่องความรู้ และการ์ด โดยมีจุดเด่นคือการเปิดกล้อง **AR** เพื่อเห็นพระบรมราชานุสาวรีย์ 3 มิติและข้อมูลลอยขึ้นมาจริง

### เป้าหมายการเรียนรู้ (Learning Objectives)
| ด้าน | สิ่งที่เด็กจะได้ |
|------|------------------|
| ความรู้ | จดจำพระนาม ยุคสมัย และพระราชกรณียกิจสำคัญของ 7 มหาราช |
| ลำดับเวลา | เข้าใจลำดับยุค สุโขทัย → อยุธยา → ธนบุรี → รัตนโกสินทร์ |
| ทักษะ | คิดวิเคราะห์ ตอบคำถาม ทำงานเป็นทีม รู้จักแพ้–ชนะ |
| ค่านิยม | ความกตัญญู ความภาคภูมิใจในชาติ (สอดคล้องแนวคิดอุทยานราชภักดิ์) |

### กลุ่มเป้าหมาย
- **หลัก:** นักเรียนประถมปลาย – มัธยมต้น (อายุ 9–14 ปี)
- **บริบทใช้งาน:** ในห้องเรียน (ครูเป็นผู้ดำเนินเกม), กิจกรรมกลุ่ม, ศูนย์การเรียนรู้/พิพิธภัณฑ์
- **โหมดเล่น:** วางแท็บเล็ตกลางโต๊ะ ผลัดกันเล่น (Pass-and-play) หรือฉายขึ้นจอโปรเจกเตอร์

## 2. เนื้อหา 7 มหาราช

พระมหากษัตริย์ทั้ง 7 พระองค์ตามพระบรมราชานุสาวรีย์ ณ อุทยานราชภักดิ์ อ.หัวหิน จ.ประจวบคีรีขันธ์ (เรียงตามลำดับเวลา):

| # | พระนาม | ยุค | พระราชกรณียกิจเด่น (ตัวอย่างเนื้อหาในเกม) |
|---|--------|-----|------------------------------------------|
| 1 | **พ่อขุนรามคำแหงมหาราช** | สุโขทัย | ประดิษฐ์อักษรไทย (ลายสือไท), ศิลาจารึกหลักที่ 1, การค้าเสรี |
| 2 | **สมเด็จพระนเรศวรมหาราช** | อยุธยา | ประกาศอิสรภาพ, สงครามยุทธหัตถี, กอบกู้เอกราช |
| 3 | **สมเด็จพระนารายณ์มหาราช** | อยุธยา | การทูตกับฝรั่งเศส, ยุคทองวรรณคดี, ความเจริญด้านดาราศาสตร์ |
| 4 | **สมเด็จพระเจ้าตากสินมหาราช** | ธนบุรี | กอบกู้เอกราชหลังเสียกรุงครั้งที่ 2, สถาปนากรุงธนบุรี |
| 5 | **พระบาทสมเด็จพระพุทธยอดฟ้าจุฬาโลกมหาราช (ร.๑)** | รัตนโกสินทร์ | สถาปนากรุงเทพฯ, ราชวงศ์จักรี, ชำระกฎหมายตราสามดวง |
| 6 | **พระบาทสมเด็จพระจอมเกล้าเจ้าอยู่หัว (ร.๔)** | รัตนโกสินทร์ | วิทยาศาสตร์–ดาราศาสตร์ (สุริยุปราคาหว้ากอ), การทูตสมัยใหม่ |
| 7 | **พระบาทสมเด็จพระจุลจอมเกล้าเจ้าอยู่หัว (ร.๕)** | รัตนโกสินทร์ | เลิกทาส, ปฏิรูปประเทศ, การรถไฟ–สาธารณูปโภค |

> 🔎 **หมายเหตุด้านความถูกต้อง:** เนื้อหาข้อเท็จจริง คำถาม และปีศักราชทั้งหมด ควรผ่านการตรวจสอบโดยครูประวัติศาสตร์/ผู้เชี่ยวชาญก่อนเผยแพร่ ไฟล์นี้ให้ตัวอย่างเชิงโครงสร้างเท่านั้น

### โครงสร้างข้อมูลของแต่ละมหาราช (Data Schema)
```jsonc
{
  "id": "king_ramkhamhaeng",
  "order": 1,
  "name": "พ่อขุนรามคำแหงมหาราช",
  "era": "สุโขทัย",
  "reignPeriod": "พ.ศ. 1822–1841 (โดยประมาณ)",
  "shortBio": "...",
  "achievements": ["ประดิษฐ์อักษรไทย", "ศิลาจารึกหลักที่ 1", "..."],
  "themeColor": "#B8860B",         // สีประจำมหาราช (ใช้กับ zone บนกระดาน)
  "portrait2D": "assets/kings/ramkhamhaeng.png",
  "arModel3D": "assets/ar/ramkhamhaeng.glb",
  "arMarkerId": "marker_king_01",
  "questions": ["q_king01_01", "q_king01_02"],  // อ้างอิงคลังคำถาม
  "funFacts": ["...", "..."]
```

## 3. กลไกเกม (Game Mechanics)

### 3.1 การเริ่มเกม
1. เลือกจำนวนผู้เล่น/ทีม (2–4)
2. เลือกหมาก (Token) — ออกแบบเป็นสัญลักษณ์ไทย เช่น ช้างศึก, เรือสำเภา, ราชรถ, พญานาค
3. ทุกคนเริ่มที่ช่อง **START (ลานอุทยานราชภักดิ์)**

### 3.2 เทิร์นการเล่น (Turn Loop)
```
เริ่มเทิร์น
   │
   ├─▶ ทอยลูกเต๋า (แตะปุ่ม/ปัดหน้าจอเพื่อทอย — มีอนิเมชัน 3D)
   │
   ├─▶ หมากเดินตามแต้ม (เดินทีละช่องพร้อมเสียง)
   │
   ├─▶ ระบบตรวจว่าหยุดที่ "ช่องประเภทใด"
   │        ├─ ช่องคำถาม   → เปิด Card คำถาม
   │        ├─ ช่องภารกิจ  → เปิด Card ภารกิจ
   │        ├─ ช่องความรู้  → แสดงเกร็ดความรู้ + ปุ่ม AR
   │        ├─ ช่องรับเหรียญ → +เหรียญทันที
   │        └─ ช่องมหาราช   → ปลดล็อกมหาราช + AR
   │
   ├─▶ ให้/หักเหรียญตามผล
   │
   └─▶ ส่งเทิร์นให้ผู้เล่นถัดไป
```

### 3.3 เงื่อนไขจบเกม (เลือกได้ในหน้าตั้งค่า)
- **โหมดรอบสนาม:** เดินครบ 1 รอบกระดาน (ผ่านครบ 7 มหาราช) → ผู้มีเหรียญมากสุดชนะ
- **โหมดเก็บแต้ม:** เล่นตามเวลาที่ครูตั้ง (เช่น 20 นาที) → นับเหรียญ
- **โหมดสะสมมหาราช:** ใครปลดล็อก "ครบ 7 มหาราช" (ตอบคำถามผ่านทุกพระองค์) เป็นคนแรกชนะ

### 3.4 ลูกเต๋า
- ลูกเต๋ามาตรฐาน 1 ลูก (แต้ม 1–6) สำหรับเด็กเล็ก / 2 ลูกสำหรับเพิ่มความเร็ว
- ทอยได้ 6 → ได้ทอยซ้ำ (โบนัส)
- อนิเมชันลูกเต๋า 3D + เสียง + haptic feedback (สั่นเบาๆ)

## 4. ประเภทช่องบนกระดาน

กระดานเป็นเส้นทางวนรอบ (loop) ประมาณ **28–35 ช่อง** แบ่งเป็น 7 โซนสี ตามมหาราชแต่ละพระองค์

| ประเภทช่อง | ไอคอน | สี | ผลเมื่อหยุด |
|-----------|-------|-----|-------------|
| 🟢 START | 🏛️ | ทอง | จุดเริ่ม / ผ่านแล้ว +เหรียญโบนัส |
| ❓ ช่องคำถาม | เครื่องหมายคำถาม | ฟ้า | ตอบคำถาม ถูก = ได้เหรียญ |
| 🎯 ช่องภารกิจ | ธง/ดาว | ส้ม | ทำภารกิจ (ใบ้คำ, เรียงลำดับ, จับคู่) |
| 💰 ช่องรับเหรียญ | เหรียญ | เหลืองทอง | ได้เหรียญทันที (ไม่มีเงื่อนไข) |
| 📖 ช่องความรู้ | หนังสือ | เขียว | อ่านเกร็ดความรู้ + ปุ่มเปิด AR |
| 👑 ช่องมหาราช | มงกุฎ | สีประจำพระองค์ | ปลดล็อกมหาราช + ดู AR อนุสาวรีย์ |
| 🎲 ช่องโชค | ลูกเต๋า/ไพ่ | ม่วง | หยิบการ์ดโชค–เหตุการณ์ (สุ่มดี/ร้าย) |
| ⏭️ ช่องพิเศษ | ลูกศร | เทา | เดินหน้า/ถอยหลัง/ข้ามตา |

### เค้าโครงกระดานแนะนำ (ตัวอย่าง)
```
[START]-[ความรู้]-[คำถาม]-[👑ร.๑มหาราช]-[เหรียญ]-[ภารกิจ]-[โชค]-...
   └──────────────── วนรอบ 7 โซน ────────────────┘
```
แต่ละโซนของมหาราช 1 พระองค์ = ~4–5 ช่อง (ความรู้ + คำถาม + ภารกิจ + ช่องมหาราช)

## 5. ระบบการ์ด (Card System)

การ์ดคือหัวใจของการเรียนรู้ แสดงเป็น **Modal เต็มจอ** เมื่อหยุดที่ช่องที่เกี่ยวข้อง

### 5.1 ประเภทการ์ด

#### 🃏 A. การ์ดคำถาม (Quiz Card)
```
┌─────────────────────────────┐
│  👑 พ่อขุนรามคำแหงมหาราช       │  ← หัวการ์ด (สีโซน)
│  ─────────────────────────  │
│  ❓ พ่อขุนรามคำแหงทรงประดิษฐ์  │
│     สิ่งใดที่คนไทยใช้จนถึง      │
│     ปัจจุบัน?                 │
│                             │
│   [ A. อักษรไทย ]            │  ← ปุ่มตัวเลือกใหญ่
│   [ B. เลขไทย   ]            │
│   [ C. ปฏิทิน    ]           │
│   [ D. ธงชาติ    ]           │
│                             │
│   ⏱️ 00:20   💰 +50          │
└─────────────────────────────┘
```
- 4 ตัวเลือก, มีตัวจับเวลา (ปิดได้), ตอบถูก = ได้เหรียญ + เอฟเฟกต์ฉลอง
- ตอบผิด = เฉลย + แสดงเกร็ดความรู้ (เรียนรู้จากความผิดพลาด ไม่ทำโทษรุนแรง)
- ระดับความยาก: ง่าย/กลาง/ยาก (ให้เหรียญต่างกัน)

#### 🎯 B. การ์ดภารกิจ (Mission Card)
รูปแบบ mini-game สั้นๆ:
- **เรียงลำดับเวลา:** ลากมหาราช/เหตุการณ์เรียงตามยุค
- **จับคู่:** จับคู่มหาราช ↔ พระราชกรณียกิจ
- **ใบ้คำ/ทาย:** ดูภาพเบลอแล้วทายพระนาม
- **True/False:** ข้อความถูกหรือผิด

#### 📖 C. การ์ดความรู้ (Knowledge Card)
- เนื้อหาสั้น อ่านง่าย (ไม่เกิน 3–4 บรรทัด) + ภาพประกอบ
- ปุ่ม **"เปิดกล้อง AR 📷"** เด่นชัด
- ไม่มีการทดสอบ — เน้นให้ซึมซับ

#### 🎲 D. การ์ดโชค–เหตุการณ์ (Chance Card)
- เชื่อมโยงเหตุการณ์ประวัติศาสตร์กับผลในเกม เช่น
  - "ยุทธหัตถีชนะ! เดินหน้า 3 ช่อง +เหรียญ"
  - "เสียกรุงครั้งที่ 2 — ถอยหลัง 2 ช่อง" (แล้วต่อด้วยการ์ดกอบกู้ของพระเจ้าตาก)

### 5.2 Data Schema ของการ์ด
```jsonc
{
  "id": "q_king01_01",
  "type": "quiz",              // quiz | mission | knowledge | chance
  "kingId": "king_ramkhamhaeng",
  "difficulty": "easy",        // easy | medium | hard
  "reward": 50,
  "timeLimitSec": 20,
  "question": "พ่อขุนรามคำแหงทรงประดิษฐ์สิ่งใด...",
  "choices": [
    { "text": "อักษรไทย", "correct": true },
    { "text": "เลขไทย", "correct": false }
  ],
  "explanation": "ทรงประดิษฐ์ลายสือไทเมื่อ พ.ศ. 1826 ...",
  "arTrigger": null            // เปิด AR หลังตอบได้หรือไม่
}
```

## 6. ระบบเหรียญและการให้คะแนน

- สกุลเงินในเกม: **"เหรียญราชภักดิ์" 🪙**
- แหล่งได้เหรียญ: ตอบคำถามถูก, ทำภารกิจสำเร็จ, ช่องรับเหรียญ, ผ่าน START
- ตัวชี้วัดความก้าวหน้า: แถบสะสม "ปลดล็อกมหาราช 3/7 👑"
- **ไม่เน้นการลงโทษ** — เด็กที่ตอบผิดยังเดินต่อได้ เพื่อรักษาแรงจูงใจ
- สรุปท้ายเกม: ตารางอันดับ + badge เช่น "นักประวัติศาสตร์น้อย", "ยอดนักตอบ"

## 7. หลักการออกแบบ UX/UI

### 7.1 Design Principles
1. **แตะง่าย นิ้วเด็ก:** ปุ่มขั้นต่ำ 48×48 dp, พื้นที่แตะเผื่อ, ระยะห่างชัด
2. **อ่านง่าย:** ฟอนต์ไทยอ่านสบาย (เช่น Sarabun, Kanit, IBM Plex Sans Thai) ขนาด ≥ 18sp
3. **ป้อนกลับทันที (Feedback):** ทุกการแตะมีเสียง + อนิเมชัน + haptic
4. **ลดข้อความ เน้นภาพ:** ใช้ไอคอน สี และภาพประกอบนำ
5. **หนึ่งหน้าจอ = หนึ่งภารกิจ:** ไม่ยัดข้อมูลจนล้น
6. **ปลอดภัยสำหรับเด็ก:** ไม่มีโฆษณา ไม่มีลิงก์ออกนอก ไม่เก็บข้อมูลส่วนตัว

### 7.2 Design Tokens (ระบบดีไซน์)
```jsonc
// สี — โทนไทยประยุกต์ ทอง–แดงชาด–ครีม
{
```

### 7.3 หลักการแยก Layout (สำคัญตามโจทย์)
> **แนวคิด:** แยก UI ของแนวตั้งและแนวนอนออกเป็น **คนละไฟล์ Layout อิสระ** ใช้ร่วมกันเฉพาะ *Logic / State / Components ย่อย* ไม่ใช้ CSS media query บีบ layout เดียวให้ยืดหด เพื่อให้ปรับแต่งแต่ละแนวได้อิสระเต็มที่

```
/screens/GameBoard/
  ├── GameBoard.logic.ts        // state, กติกา, ใช้ร่วมกัน
  ├── GameBoard.portrait.tsx    // Layout แนวตั้ง (อิสระ)
  ├── GameBoard.landscape.tsx   // Layout แนวนอน (อิสระ)
  └── index.tsx                 // เลือก layout ตาม orientation
```
```tsx
// index.tsx — ตัวเลือก layout ตามการหมุนจอ
export default function GameBoard(props) {
  const orientation = useOrientation();     // 'portrait' | 'landscape'
  return orientation === 'portrait'
    ? <GameBoardPortrait {...props} />
    : <GameBoardLandscape {...props} />;
}
```

## 10. ระบบ AR (Augmented Reality)

> จุดขายของสื่อการสอนนี้ — เด็กเปิดกล้องแล้วเห็นพระบรมราชานุสาวรีย์ 3 มิติ + ข้อมูลลอยขึ้นมา

### 10.1 รูปแบบการใช้งาน AR (เลือกได้ 3 โหมด)

#### โหมด A — **Marker-based AR (แนะนำสำหรับห้องเรียน)** ⭐
- แจก **การ์ดภาพ (AR Marker)** ของแต่ละมหาราช (พิมพ์แจก หรืออยู่ในคู่มือ)
- เด็กส่องกล้องไปที่การ์ด → โมเดล 3 มิติของอนุสาวรีย์ผุดขึ้น + ป้ายข้อมูลลอย
- ข้อดี: เสถียร ใช้ในห้องเรียนได้จริง ไม่ต้องพื้นที่กว้าง
- เทคโนโลยี: AR.js / MindAR (บนเว็บ) หรือ Vuforia (บน native)

#### โหมด B — **Markerless / Plane AR**
- วางโมเดลบนพื้น/โต๊ะที่กล้องตรวจจับได้ → เด็กเดินรอบดูได้ 360°
- เทคโนโลยี: ARKit (iOS) / ARCore (Android) / WebXR

#### โหมด C — **In-app 3D Viewer (Fallback ไม่ใช้กล้อง)**
- อุปกรณ์ไม่รองรับ AR → แสดงโมเดล 3 มิติในกรอบให้หมุน–ซูมดูแทน
- **สำคัญ:** ต้องมี fallback นี้เสมอ เพื่อไม่ให้เด็กที่ใช้เครื่องรุ่นเก่าเสียโอกาส

### 10.2 Flow การเปิด AR
```
ช่องความรู้/ช่องมหาราช
   │
   ├─▶ ปุ่ม "เปิดกล้อง AR 📷"
   │
   ├─▶ ตรวจสิทธิ์กล้อง (Permission)
   │      ├─ อนุญาต → เข้าโหมด AR
   │      └─ ปฏิเสธ/ไม่รองรับ → เปิด 3D Viewer (โหมด C)
   │
   ├─▶ [AR Scene]
   │      • โมเดลอนุสาวรีย์ 3D (.glb/.usdz)
   │      • ป้ายข้อมูลลอย (พระนาม, ยุค, พระราชกรณียกิจ)
   │      • ปุ่ม 🔊 ฟังเสียงบรรยาย
   │      • ปุ่ม 📸 ถ่ายภาพคู่ (เพิ่มความสนุก/แชร์ในห้อง)
   │
   └─▶ ปุ่ม "กลับเข้าเกม" → รับเหรียญโบนัสจากการเรียนรู้ AR
```

### 10.3 UI ในฉาก AR
```

║ [← กลับ]                    [🔊]  ║
║                                   ║
║          (ภาพจากกล้องจริง)         ║
║                                   ║
║            🗿 โมเดล 3D             ║
║         พระบรมราชานุสาวรีย์         ║
║           ┌─────────────┐         ║
║           │👑 ร.๕        │         ║  ← ป้ายข้อมูลลอย (billboard)
║           │เลิกทาส พ.ศ.  │         ║     หันหน้าเข้าหากล้องเสมอ
║           │2448          │         ║
║           └─────────────┘         ║
║                                   ║
```

### 10.4 ข้อกำหนดทางเทคนิค AR
| หัวข้อ | รายละเอียด |
|--------|-----------|
| ฟอร์แมตโมเดล | `.glb` (Android/Web), `.usdz` (iOS Quick Look) |
| ขนาดโมเดล | ควบคุม < 5–10 MB/ตัว เพื่อโหลดไว, polygon ต่ำ (LOD) |
| แสง | ใช้ ambient + directional จำลองแดด, เปิด shadow อ่อนๆ |
| ป้ายข้อมูล | Billboard หันเข้ากล้อง, ตัวหนังสือใหญ่, มีพื้นหลังโปร่งแสง |
| เสียง | ไฟล์บรรยายสั้น 20–30 วินาที/พระองค์ |
| Performance | ปิด AR อัตโนมัติเมื่อเฟรมเรตต่ำ, มี loading state |
| Privacy | ภาพจากกล้อง **ไม่อัปโหลด** ประมวลผลบนเครื่องเท่านั้น |

## 13. Accessibility สำหรับเด็ก

- **ขนาดสัมผัส:** ปุ่มทุกปุ่ม ≥ 48dp, ระยะห่างชิ้นแตะ ≥ 8dp
- **คอนทราสต์:** ตัวอักษรผ่าน WCAG AA (อัตราส่วน ≥ 4.5:1)
- **เสียงประกอบข้อความ:** มีปุ่มอ่านออกเสียง (TTS) ช่วยเด็กที่อ่านช้า
- **ไม่กดดันเวลา:** ตัวจับเวลาปิดได้ / ครูตั้งค่าได้
- **สีไม่ใช่สื่อเดียว:** ใช้ไอคอน + ข้อความคู่กับสีเสมอ (เผื่อตาบอดสี)
- **โหมดครู (Teacher Mode):** ตั้งจำนวนคำถาม, ระดับความยาก, เวลา, เปิด/ปิด AR
- **ออฟไลน์:** เล่นได้โดยไม่ต้องต่อเน็ต (เนื้อหาฝังในแอป) — เหมาะห้องเรียนที่เน็ตไม่เสถียร
