# 📷 แนวทางทำ AR จริง — บอร์ดเกม 7 มหาราช

> บันทึกไอเดียการทำ AR ที่ "ไม่ยากเกินไป" สำหรับสื่อการสอนบนแท็บเล็ต (เว็บ React + Vite PWA)
> อัปเดต: 2026-07-04

---

## 🎯 สรุปสั้น (TL;DR)
- **แนะนำที่สุด:** ใช้ **`<model-viewer>` ของ Google** — โค้ดน้อยมาก ไม่ต้องเขียน computer vision, OS จัดการ AR ให้, ได้ทั้ง 3D viewer + ปุ่ม AR + ป้ายข้อมูลลอย (hotspot)
- **ถ้าอยากได้ overlay ผ่านกล้องแบบเต็มตัว:** ใช้ **MindAR** (marker-based) พิมพ์การ์ดแจกเด็ก
- **ปัญหาเดียวที่เหลือจริง ๆ:** หา **โมเดล 3D (.glb/.usdz)** ของพระบรมราชานุสาวรีย์ → วิธีถูกสุดคือ **สแกนรูปปั้นจริงที่อุทยานราชภักดิ์ด้วยมือถือ**

## 🥇 ทางเลือกที่ 1: `<model-viewer>` (Google) — ง่ายสุด ⭐ แนะนำ

Web component ตัวเดียว ใส่ไฟล์ `.glb` แล้วได้ 2 อย่างในตัว:
1. **3D Viewer** — หมุน/ซูมพระบรมราชานุสาวรีย์ในหน้าเว็บ (ใช้แทน fallback เดิมได้เลย)
2. **ปุ่ม "ดูในพื้นที่จริง"** — กดบนแท็บเล็ตแล้วระบบเรียก **AR ของ OS เอง**
   - Android → Scene Viewer (ARCore) ใช้ `.glb`
   - iOS → Quick Look ใช้ `.usdz`
   - เด็กเห็นรูปปั้นตั้งบนโต๊ะผ่านกล้องจริง

### ตัวอย่างโค้ด
```html
<model-viewer
  src="rama5.glb"
  ios-src="rama5.usdz"
  ar
  ar-modes="scene-viewer quick-look webxr"
  camera-controls
  auto-rotate
  shadow-intensity="1">
  <!-- ป้ายข้อมูลลอย = hotspot ติดกับโมเดล (ตรงตามดีไซน์ "ข้อมูลลอยขึ้นมา") -->
  <button slot="hotspot-1" data-position="0 1.6 0" data-normal="0 1 0">
    👑 ร.๕ · เลิกทาส พ.ศ. 2448
  </button>
</model-viewer>
```

### ข้อดี
- ✅ **ไม่ต้องเขียนกล้อง / ไม่ต้องมี marker / ไม่ต้องจับ plane เอง** — OS จัดการ AR ให้
- ✅ ข้ามแพลตฟอร์ม (Android + iOS + เดสก์ท็อป)
- ✅ ติดตั้งผ่าน npm `@google/model-viewer` → self-host → **ทำงานออฟไลน์ในห้องเรียน**
- ✅ รองรับ **hotspot ป้ายลอย** พอดีกับดีไซน์
- ✅ Fallback ในตัว: อุปกรณ์ไม่รองรับ AR ก็ยังหมุนดู 3D ได้

### ข้อจำกัด
- ⚠️ ตอนเข้าโหมด AR จริงจะ "เด้งไป" viewer ของ OS ชั่วคราว (ไม่ได้อยู่ในหน้าเว็บเรา 100%)
- ⚠️ ป้าย hotspot ปรับแต่งได้จำกัดกว่าการเขียน 3D scene เอง

## 🥈 ทางเลือกที่ 2: MindAR (marker-based) — "ว้าว" กว่า แต่แรงขึ้นนิด

พิมพ์ **การ์ดรูปมหาราช (marker)** แจกเด็ก → ส่องกล้อง → โมเดล 3D ผุดทับการ์ดใน**ภาพกล้องจริง และค้างอยู่ในหน้าเว็บเรา** (ไม่เด้งออก OS) ทำงานในเบราว์เซอร์ล้วน (WebAR)


### ข้อดี
- ✅ Overlay ผ่านกล้องจริงเต็มตัว อยู่ในแอปเรา 100%
- ✅ ทำงานในเบราว์เซอร์ ออฟไลน์ได้ (หลัง cache assets)
- ✅ เหมาะห้องเรียน — การ์ด marker เป็นสื่อจับต้องได้

### ข้อจำกัด / งานที่ต้องทำเพิ่ม
- ต้องเขียน Three.js scene เล็กน้อย (วางโมเดล + ป้าย + แสง)
- ต้องคอมไพล์ภาพ marker เป็นไฟล์ `.mind`
- ต้องพิมพ์การ์ด marker แจก
- ไวต่อแสง/มุมกล้อง

## 🥉 ทางเลือกที่ 3: WebXR markerless (plane detection) — ยากขึ้น

วางรูปปั้นบนพื้น/โต๊ะ เดินดูรอบ 360° immersion สูงสุด แต่:
- ⚠️ device support ยังไม่ทั่วถึง
- ⚠️ โค้ดมากขึ้น, เด็กเล็กควบคุมยากกว่า

→ **ยังไม่แนะนำสำหรับเฟสแรก** (เก็บเป็น future upgrade)

## 🗿 ปัญหาสำคัญ: หาโมเดล 3D มาจากไหน (ราคาถูก)

เพราะรูปปั้น **มีอยู่จริงที่อุทยานราชภักดิ์ อ.หัวหิน** → มีวิธีถูก ๆ หลายทาง:

| วิธี | เครื่องมือ | ต้นทุน | หมายเหตุ |
|------|-----------|--------|----------|
| **📱 สแกนรูปปั้นจริง (Photogrammetry)** ⭐ | Polycam / RealityScan (แอปมือถือฟรี) เดินถ่ายรอบรูปปั้น → export `.glb` | ~0฿ แค่ไปถ่าย | ตรงกับสถานที่จริง + เป็นกิจกรรมที่เข้าธีม |
| 🤖 สร้างด้วย AI จากภาพ | Meshy / Luma Genie (ใส่รูปพระบรมสาทิสลักษณ์) | ฟรี–ถูก | เร็ว แต่ต้องตรวจความเหมาะสม |
| 🎨 โมเดล low-poly / placeholder | จ้างทำ หรือดาวน์โหลดรูปปั้นแบบเรียบ | ต่ำ | ใช้ระหว่างรอโมเดลจริง |

> ⚠️ **ข้อควรระวังด้านความเหมาะสม:** โมเดล 3D ของพระบรมราชานุสาวรีย์ต้องทำอย่าง **สมพระเกียรติ** และผ่านการตรวจสอบก่อนเผยแพร่ (สอดคล้องกับหมายเหตุใน `DESIGN.md`)

### ฟอร์แมตไฟล์ที่ต้องเตรียม (ต่อ 1 พระองค์)
- `.glb` — สำหรับ Android / Web / 3D viewer
- `.usdz` — สำหรับ iOS Quick Look
- ขนาด < 5–10 MB/ตัว, polygon ต่ำ (โหลดไว), มี LOD ถ้าทำได้

## 🔌 แผนการนำไปใส่ (Integration Plan)

โปรเจกต์เตรียม field ไว้แล้วใน `src/data/kings.json`:
```jsonc
"arModel3D": "assets/ar/rama5.glb",     // Android/Web
"arModelIOS": "assets/ar/rama5.usdz",   // iOS
"arMarkerId": "marker_king_07"          // สำหรับ MindAR (ถ้าเลือกทาง 2)
```

### ขั้นตอน (ทางเลือกที่ 1 — model-viewer)
1. `npm i @google/model-viewer` (self-host เพื่อออฟไลน์)
2. แก้ `src/components/ARLauncher.tsx` — แทน placeholder ด้วย `<model-viewer>` อ่าน path จาก `king.arModel3D` / `king.arModelIOS`
3. ใส่ hotspot ป้ายลอยจาก `king.name` + `king.achievements`
4. วางไฟล์โมเดลใน `src/assets/ar/` (เริ่มด้วย placeholder .glb ก่อน แล้วสลับเป็นรูปปั้นจริง — แค่เปลี่ยน path)
5. เคารพ `settings.arEnabled` (Teacher Mode) ที่มีอยู่แล้ว

### ทางเลือกที่ 2 — MindAR (ถ้าต้องการ overlay กล้องเต็มตัว)
1. `npm i mind-ar three`
2. คอมไพล์ภาพ marker → `.mind` (MindAR image compiler)
3. เขียน AR scene (กล้อง + anchor + โมเดล + billboard label)
4. พิมพ์การ์ด marker 7 ใบแจก

## ✅ คำแนะนำสรุป
1. **เริ่มด้วย `<model-viewer>`** — ได้ AR ใช้งานจริงบนแท็บเล็ตเร็วที่สุด ด้วยโค้ดน้อยสุด
2. วาง **placeholder .glb** ก่อน เพื่อทดสอบ flow → ค่อยสลับโมเดลรูปปั้นจริงทีหลัง
3. ได้โมเดลจริงจาก **การสแกนรูปปั้นที่อุทยานราชภักดิ์ด้วยมือถือ** (ถูก + ตรงธีม)
4. ถ้าภายหลังอยากได้ "ว้าว" แบบ overlay กล้อง → เพิ่ม **MindAR** เป็น mode ที่ 2
5. **ตรวจความสมพระเกียรติของโมเดล** ก่อนเผยแพร่เสมอ
